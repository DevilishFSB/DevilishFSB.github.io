Migrando para o Octopress, Parte Abortada

Published 21/09/2012 | By Demente

Sim, eu desisti de migrar para o Octopress. Percebi que essa migração ia exigir uma mudança drástica não apenas no ritmo das postagens, mas também na dinâmica do Blog - além de me fazer aprender Ruby quando ainda estou derrapando no PHP. Ao invés, pretendo usar uma instalação local do WordPress com o WP Static HTML Output.

Porém, se você estiver interessado em como migrar postagens do WordPress para Jekyll (o núcleo do Octopress), desça até o final da postagem.

Voltando ao assunto: eu não havia escrito que esse método demora pra caramba? Bem, alguns dias atrás ele foi atualizado e agora o site leva cerca de 2-3 minutos para ser convertido para uma versão estática (contra 1 Hora antes da atualização). O Octopress leva 15 segundos para fazer a mesma tarefa, mas… o WordPress + WP static é mais confortável por hora.

A minha estratégia é escrever as postagens num WordPress instalado em meu computador, e então usar o WP Static para gerar uma versão estática do Blog pronta para ser subida ao meu servidor de conteúdo. Já ajustei os comentários via Disqus, faltando apenas configurar as páginas de erro, mas fora isso o Blog mostrou-se plenamente funcional nos testes iniciais.

Sim, isso também é uma mudança drástica, mas é bem menos trabalhosa do que aprender Ruby.

Para compensar, estou escrevendo uma apostila que ensina, passo a passo, como rodar o WordPress localmente e como configurá-lo para gerar um blog estático funcional, como o próprio Masmorra Descuidada é agora.

Até a próxima e, para quem veio ao Blog procurando como transferir postagens do WordPress para Octopress, deixo como consolo o método tosco a seguir!

Exportar Postagens (toscamente) do WordPress para Jekyll/Octopress

– Baixe o Programa WpXml2Jekyll (incrivelmente tosco, mas funcional);

– Baixe um arquivo de exportação do seu Blog WordPress;

– Abra o arquivo chamado “wpXml2Jekyll.exe”;

– Clique em “Select Xml” e abra o arquivo de exportação previamente baixado;

– Clique em “Save Posts” e escolha uma pasta para salvar suas postagens.

Daí, pegue os arquivos .md gerados para alimentar sua instalação do Jekyll/Octopress.

Migrando para o Octopress, Parte 1

Published 16/09/2012 | By Demente

Conteúdo Geek à vista, tenham cuidado.

Após a pesquisa que andei fazendo nas últimas semanas, eu decidi migrar – de alguma forma – do WordPress para o Octopress. Isso vai exigir um pouco de trabalho da minha parte, e isso pode afetar ainda mais a freqüencia de atualização do blog, então é melhor explicar logo a situação.

A verdade é que a quantidade de visitas ao Blog vêm crescido ultimamente, o que significa mais trabalho para mantê-lo veloz (a parte simples) e a necessidade de contratar uma hospedagem mais potente (a parte difícil).

Tá, mas porque estou decepcionado com o WordPress? Pois é, não estou. O problema é de ordem financeira mesmo: atualmente, pago o equivalente a pouco mais de 12 reais por mês para manter o Blog. É pouco, realmente, mas se lembrarmos que eu não monetizo o Blog, a conta acaba pesando no final do ano – o que pode piorar se eu tiver que sair de meu plano de hospedagem atual para outro mais caro.

Gostaria de falar um pouco sobre Cache: como o WordPress é um site dinâmico, toda vez que uma visita ou troca de página é feita, o servidor “redesenha” a página visitada. O Cache serve para guardar a página já “desenhada” e serví-la aos visitantes seguintes.

O problema é que sistemas de Cache possuem sérios limites, como bugs nos comentários (tanto minha caixa de spam quanto a caixa de comentários válidos fica vazia, ao mesmo tempo que vejo um ou outro chingamento no meu email…) ou mal funcionamento de plugins (o WP Super Cache me deu muita dor de cabeça no passado).

É nesse ponto que cheguei a usar o plugin WP Static HTML Output, combinado com comentários via Disqus (será que alguém lembra?). Porém esse plugin levava quase 1 hora para converter meu Blog para o formato estático, e isso era chato.

Então pensei: não existe uma forma de criar todo o conteúdo do site estaticamente e então jogar toda a bagaça no servidor? Fiz uma longa pesquisa e encontrei o Octopress.

Basicamente, o Octopress pega um monte de arquivos de texto e os converte em arquivos estáticos prontos para serem subidos num Servidor ou em serviços especializados como o Github Pages (para a qual estou pensando em migrar). Isso não é tão prático quanto ter que lidar, por exemplo, com o Plugin W3 Total Cache, porém é a opção com maior custo-benefício dentre as opções que pesquisei por uma razão simples.

Latência.

Mesmo mantendo um Cache, leva mais de dois segundos até o WordPress do meu servidor determinar qual página deve mostrar ao visitante, e isso é um problema quando você quer, por exemplo, manter uma boa posição no Google. Mas não pense que isso é problema só do meu servidor: chequei vários servidores que forneciam períodos de teste (Demo) e o problema persistia. A melhor forma de mitigar esse problema sem abandonar o WordPress é contratar uma hospedagem mais potente.

“Porra, Demente! Mas você não pode esperar muito de uma hospedagem barata!” Dizem os críticos. Eles têm razão. Por isso que vou passar a usar arquivos estáticos via Octopress, pois arquivos estáticos são visualizados MUITO mais rápido que qualquer sistema de cache dinâmico.

Como eu pretendo fazer isso? Bem, isso fica para a próxima parte desta série.
